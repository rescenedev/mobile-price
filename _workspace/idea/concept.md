# ratsaver — 콘셉트 검증

## 한줄 가치제안
"내 사용량 기준 가장 싼 요금제를, 로그인·가입 강요 없이 3초 안에." — 알뜰폰·통신사 요금제를 조건 필터로 비교하고 절약액을 즉시 계산해주는 가볍고 빠른 공개 비교 웹.

## 콘셉트 검증 (왜 지금, 왜 이게 통하나)

**1. 시장 수요는 검증 끝 (de-risked).**
- 모요(moyoplan.com)는 2021년 출시, 2022년 3월 기준 누적 이용자 45만 명을 넘기며 카카오벤처스 등에서 시드 투자를 유치했다. "요금제 비교"라는 JTBD 자체에 충분한 수요가 있음이 이미 입증됨. (출처: 아시아경제 2022-03)
- 알뜰폰허브(한국정보통신진흥협회 운영), 폰비, 세모통, 알닷(LGU+ MVNO) 등 다수 사업자가 존재 — 즉 "사람들이 실제로 비교하러 온다"는 행동이 검증된 카테고리.

**2. 2026년 정책 변수가 비교 수요를 재점화한다.**
- 2026년 2월 "1만원대 데이터 무제한" 통신비 인하 정책 추진 보도(다음뉴스 2026-02-20). 요금제 구조가 흔들리는 시기 = 소비자가 "지금 갈아타야 하나?"를 가장 많이 검색하는 시기. 비교 도구의 트래픽 기회.

**3. 핵심 페인포인트: 기존 비교 사이트는 "무겁고, 가입을 강요한다".**
- 모요/세모통 등은 결국 개통 수수료가 수익원이라 UX가 "가입 깔때기(funnel)"로 설계됨 — 광고, 제휴 카드, 가입 CTA가 비교 본질을 가린다.
- 사용자가 진짜 원하는 건 단순 비교 + 절약액 숫자인데, 그걸 보려면 스크롤·팝업·로그인 유도를 통과해야 한다.

## ratsaver의 차별점 (wedge)

| 축 | 기존 비교 사이트 | ratsaver |
|----|------------------|----------|
| 진입 마찰 | 로그인/가입 깔때기, 팝업 | **무인증·무팝업, 즉시 사용** |
| 속도 | 외부 API·광고 SDK로 무거움 | **시드 데이터 기반 SSG/ISR → LCP<1.5s 목표** |
| 핵심 가치 | "가입하러 가기"가 1순위 | **"절약액 숫자"가 1순위** — 절약 계산기가 히어로 |
| 데이터 정직성 | 프로모션가(7개월) 강조, 이후 정가 숨김 | **프로모션가 + 종료 후 정가 병기**(throttle·약정 명시) |

> **핵심 wedge 한 줄**: "가입시키려는 비교 사이트"가 아니라 "정직하게 비교만 해주는 빠른 계산기". 수익(제휴 CTA)은 비교 경험을 해치지 않는 자리에만 비워둔다.

## MVP 4대 기능과 가치 연결
1. **조건 필터 검색** → 탐색 진입점 (망·데이터·가격대로 좁히기)
2. **상세 + 비교 테이블** → 의사결정 보조 (프로모션가 vs 정가, throttle 명시)
3. **맞춤 추천** → 선택 피로 제거 (사용량 입력 → 점수 매칭)
4. **절약액 계산기** → 전환 트리거이자 히어로 (현재 요금 - 추천 요금 = 절약액)

## 규제/PII 플래그 (후속 Phase 대비)
- **무인증·무계정 → PII 수집 없음.** 사용자가 입력하는 사용량/현재요금은 **서버 저장 없이 클라이언트 계산**으로 처리 가능 → 개인정보보호법 노출 최소.
- **통신/금융 제휴는 MVP 비범위** — 추후 제휴 CTA 도입 시 전자상거래·통신판매중개 고지 의무 검토 필요(현 단계 플래그만).
- 요금제 데이터는 공개 정보 큐레이션이나, **"최신성·정확성" 면책 고지** 필요(요금은 수시 변동). data-model-notes.md의 `lastVerifiedAt` 필드로 대응.

## 실현 가능성 결론
- **읽기 중심 + 무인증 + 시드 데이터 50~150건** → Cloudflare Workers + D1 + KV/Cache에 거의 완벽히 적합. 쓰기·실시간·백그라운드 잡 없음.
- 추천/절약 계산은 클라이언트 또는 edge 함수로 충분(150건 스코어링은 마이크로초 단위).
- **기술 리스크 낮음, 시장 리스크 낮음(검증된 카테고리), 차별화는 UX·정직성·속도에서 확보.**
